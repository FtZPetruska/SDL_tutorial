SDL Tutorial

Setting up the Xcode project

In this section, we will cover the creation of C++ projects in Xcode and linking the SDL2 dependencies.

Creating a project

Open Xcode and click "Create a new project".

The only template that allows using C/C++ is Command Line Tool under the macOS tab. Select this one, give it a name and set the language to either C or C++. The other options can be set at your own discretion. Finally, select the directory you want to store the project in.

Configuring the Project

In the file tree, click on the project (top-level item) to reveal the configuration menu. On the left side of the menu, click on the project name and on build settings. Scroll down to (or search for) macOS Deployment Target, this is the minimum macOS version you want your application to run on. At the time of writing, the minimum version for base SDL is 10.6 (Snow Leopard), but while you can force your project to target the same version it may fail linking. As such, if compatibility is a concern, pick the oldest version that is supported by your version of Xcode (i.e. present in the drop-down menu).

Now select your target. In the Build Phases tab, expand the Link Binary With Libraries section. This is the place you want to list all the frameworks you want to link. To add the SDL2 frameworks, click on the + symbol, at the bottom click on "Add others..." and "Add files..." and navigate to /Library/Frameworks (you can open a navigation bar with CMD+Shift+G). You can select multiple frameworks at once.

Lastly, by default you will not be able to dynamically load some SDL2 libraries due to code signing. To fix this you can do either of these actions:

  • in Signing & Capabilities, tick Disable Library Validation,
  • or, in Build Settings, set Enable Hardened Runtime to No,
  • or, in General, for each framework pick the option Embed & Sign (permitted by SDL2's zlib license).

Testing the project

In order to test the project, go back into the file tree and click on the file called main. We will replace its content with a program that initializes all the libraries, prints out their version and shuts them all down.

Test code 
#include <SDL2/SDL.h>
#include <SDL2_image/SDL_image.h>
#include <SDL2_mixer/SDL_mixer.h>
#include <SDL2_net/SDL_net.h>
#include <SDL2_ttf/SDL_ttf.h>

/**
 * @brief Initializes every library.
 */
void init_all(void) {
  if (0 > SDL_Init(SDL_INIT_VIDEO)) {
    SDL_Log("%s\n", SDL_GetError());
    return;
  }

  if (0 > IMG_Init(IMG_INIT_PNG)) {
    SDL_Log("%s\n", IMG_GetError());
    return;
  }

  if (0 > Mix_Init(MIX_INIT_MP3)) {
    SDL_Log("%s\n", Mix_GetError());
    return;
  }

  if (0 > SDLNet_Init()) {
    SDL_Log("%s\n", SDLNet_GetError());
    return;
  }

  if (0 > TTF_Init()) {
    SDL_Log("%s", TTF_GetError());
    return;
  }

  SDL_Log("Every library was initialized successfully.\n");
}

/**
 * @brief Quits all libraries.
 */
void quit_all(void) {
  TTF_Quit();
  SDLNet_Quit();
  Mix_Quit();
  IMG_Quit();
  SDL_Quit();
}

/**
 * @brief For each libraries, logs the installed version.
 */
void log_libraries_version(void) {
  SDL_version sdl_compiled_version;
  SDL_VERSION(&sdl_compiled_version);
  SDL_version sdl_linked_version;
  SDL_GetVersion(&sdl_linked_version);
  SDL_Log("SDL:\n"
          "\tCompiled against: %d.%d.%d\n"
          "\tLinked against: %d.%d.%d\n",
          sdl_compiled_version.major, sdl_compiled_version.minor,
          sdl_compiled_version.patch, sdl_linked_version.major,
          sdl_linked_version.minor, sdl_linked_version.patch);

  SDL_version img_compiled_version;
  SDL_IMAGE_VERSION(&img_compiled_version);
  const SDL_version *img_linked_version = IMG_Linked_Version();
  SDL_Log("SDL Image:\n"
          "\tCompiled against: %d.%d.%d\n"
          "\tLinked against: %d.%d.%d\n",
          img_compiled_version.major, img_compiled_version.minor,
          img_compiled_version.patch, img_linked_version->major,
          img_linked_version->minor, img_linked_version->patch);

  SDL_version mix_compiled_version;
  SDL_MIXER_VERSION(&mix_compiled_version);
  const SDL_version *mix_linked_version = Mix_Linked_Version();
  SDL_Log("SDL Mixer:\n"
          "\tCompiled against: %d.%d.%d\n"
          "\tLinked against: %d.%d.%d\n",
          mix_compiled_version.major, mix_compiled_version.minor,
          mix_compiled_version.patch, mix_linked_version->major,
          mix_linked_version->minor, mix_linked_version->patch);

  SDLNet_version net_compiled_version;
  SDL_NET_VERSION(&net_compiled_version);
  const SDLNet_version *net_linked_version = SDLNet_Linked_Version();
  SDL_Log("SDL Net:\n"
          "\tCompiled against: %d.%d.%d\n"
          "\tLinked against: %d.%d.%d\n",
          net_compiled_version.major, net_compiled_version.minor,
          net_compiled_version.patch, net_linked_version->major,
          net_linked_version->minor, net_linked_version->patch);

  SDL_version ttf_compiled_version;
  TTF_VERSION(&ttf_compiled_version);
  const SDL_version *ttf_linked_version = TTF_Linked_Version();
  SDL_Log("SDL TTF:\n"
          "\tCompiled against: %d.%d.%d\n"
          "\tLinked against: %d.%d.%d\n",
          ttf_compiled_version.major, ttf_compiled_version.minor,
          ttf_compiled_version.patch, ttf_linked_version->major,
          ttf_linked_version->minor, ttf_linked_version->patch);
}

int main(int argc, const char * argv[]) {
  init_all();
  log_libraries_version();
  quit_all();

  return 0;
}

After pasting the code and saving the file, press CMD+R to run the executable, if no error shows up: congratulations, you are ready to get coding!
