MSYS2

So, you are on Windows and because of [reasons], you cannot use MSVC. Your best bet in that case is to use MSYS2 to manage your development libraries and compiler.

Installing a Toolchain

MSYS2 comes with the same pacman as Arch Linux. Firstly, we want to install a compiler, do note the compiler you want to use as the development libraries' name will be based upon it. The available choices (for x64) are:

  • mingw-w64-x86_64-gcc, GCC, uses MSVCRT and GNU libstdc++
  • mingw-w64-ucrt-x86_64-gcc, GCC, uses UCRT and GNU libstdc++
  • mingw-w64-clang-x86_64-clang, LLVM Clang, uses UCRT and LLVM libc++

The differences between the environments are described here.

For each package listed below the prefix (mingw-w64-x86_64/mingw-w64-ucrt-x86_64/mingw-w64-clang-x86_64) will be replaced by X, replace it by the one used by the compiler you chose.

Install pkg-config and GNU Make:

pacman -S X-pkg-config X-make

Installing SDL2

There are many versions of each library, make sure to download the one meant for the toolchain you chose.

Here are each package and what they provide:

  • X-SDL2, provides the sdl2 pkg-config target
  • X-SDL2_image, provides the SDL2_image pkg-config target
  • X-SDL2_mixer, provides the SDL2_mixer pkg-config target
  • X-SDL2_net, provides the SDL2_net pkg-config target
  • X-SDL2_ttf, provides the SDL2_ttf pkg-config target
